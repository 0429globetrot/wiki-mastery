# /wiki-init — 初始化 Wiki 知识库

首次使用本知识库时运行此命令。自动安装依赖、通过问答配置知识地图、展示使用指南。

## 执行流程

### 第一步：检查并安装 markitdown

运行以下命令检查 markitdown 是否已安装：

```bash
bash .claude/scripts/install-markitdown.sh
```

如果已安装会跳过，未安装会自动创建 venv 并安装。

### 第二步：检查目录结构

确认以下目录存在（不存在则创建）：
- source/
- raw/
- Clippings/
- learning/
- wiki/entities/
- wiki/concepts/
- wiki/sources/
- wiki/comparisons/
- wiki/meta/
- attachments/

### 第三步：知识地图问答

依次向用户提问以下问题，根据回答生成 `overview.md`：

**问题 1：** 这个知识库是关于什么领域的？（例如：计算机科学、人工智能、前端开发、机器学习、游戏开发等）

**问题 2：** 你目前的学习目标是什么？（例如：系统学习 C#、掌握 React 全栈开发、准备某个考试等）

**问题 3：** 你已经有哪些基础？（例如：有 Python 基础、零基础、了解基本数据结构等）

**问题 4：** 你计划学习的资料有哪些？（可以列几个，也可以后续再添加）

收集完回答后，生成 `overview.md`，格式如下：

```markdown
# 知识地图

## 学习领域

[用户回答的领域]

## 学习目标

[用户回答的目标]

## 当前基础

[用户回答的基础]

## 知识分类

<!-- 根据领域自动推断分类，例如前端开发可分为：HTML/CSS、JavaScript、框架、工程化、性能优化等 -->

### [分类1]

### [分类2]

### [分类3]

## 学习资源

| 资料 | 类型 | 状态 |
|------|------|------|
| [用户提到的资料] | 书籍/视频/文章 | 未开始 |

## 知识关联

<!-- 学习过程中发现的不同主题之间的关系 -->

## 待探索

<!-- 有兴趣但还没开始的主题 -->
```

**注意：** 如果 overview.md 已有内容（非空模板），询问用户是否要覆盖还是跳过此步骤。

### 第四步：输出欢迎信息

向用户展示：

```
🎉 Wiki 知识库初始化完成！

📁 目录结构：
  source/      ← 把原始资料（PDF、docx 等）放这里
  raw/         ← 转换后的 markdown（自动生成）
  learning/    ← 学习过程文件（自动生成）
  wiki/        ← 沉淀后的知识（自动生成）

📝 快速开始：
  1. 把学习资料放入 source/
  2. 使用 /wiki-source [文档名] 导入资料

📋 可用命令：
  /wiki-source [文档名]           重量资料导入（书籍等）
  /wiki-ingest                    轻量资料导入（文章、网页）
  /wiki-learning-unit [文档] [单元]  苏格拉底问答学习
  /wiki-learning-over [文档名]    全书总结 + 沉淀到 wiki
  /wiki-query [问题]              知识查询
  /wiki-lint                      wiki 健康检查

💡 提示：所有命令都支持中文触发，例如说「导入书」「开始学习」等。
```

# /wiki-learning-unit — 苏格拉底问答学习

用户自学完一个单元后，通过苏格拉底式问答加深理解，完成后自动沉淀笔记。

## 触发方式

用户说 `/wiki-learning-unit [文档名] [单元名]`，或说「学习xxx的xxx章」。

## 执行流程

### 第一步：读取上下文

1. 读取 `learning/[文档名]/progress.md` 确认当前状态
2. 读取 `learning/[文档名]/骨架.md` 找到该单元的所有知识点
3. 读取 `learning/[文档名]/notes/` 中已有的笔记（如有）
4. 如果 progress.md 显示该单元已完成，提示用户是否要重新学习

### 第二步：提示自学

告诉用户：
- 该单元包含哪些知识点
- 提示用户阅读 raw/ 中对应的章节内容
- 告诉用户准备好了就说一声

### 第三步：苏格拉底问答

用户确认自学完成后，开始逐知识点提问。

**提问原则：**
- 深度：理解 + 应用 + 辨析
- 不要一次问太多，每个知识点 1-2 个问题
- 问题要具体，不要问「你理解了吗」这种空泛的问题

**答对时：**
- 简短肯定（不要过度夸奖）
- 可以追问更深层的问题（应用/辨析层面）
- 然后进入下一个知识点

**答错时 — 坚持不给直接答案：**
- 第一次：换个角度重新提问，或把问题拆小
- 第二次：给一个暗示或提示，引导思考方向
- 第三次：给一个更具体的提示，但仍然不说答案
- 如果持续卡住，可以给一个小例子来启发
- 最终引导用户自己想出来

**标记机制：**
- 记录用户答错次数和卡住的知识点
- 这些是后续沉淀的重点内容

### 第四步：自动沉淀

所有知识点问答完毕后，不需要用户额外触发，直接沉淀：

1. 写入 `learning/[文档名]/notes/[单元名].md`，格式：
   ```markdown
   # [单元名] 学习笔记

   ## 知识点总结
   （每个知识点的理解，用自己的话）

   ## 重点和难点
   （答错的、反复卡住的知识点，展开详细解释）

   ## 与其他知识点的联系
   （这个单元内部知识点之间的关系）

   ## 遗留问题
   （还有疑问的地方）
   ```

2. 更新 `learning/[文档名]/progress.md`（标记该单元完成）

### 第五步：向用户汇报

告诉用户：
- 单元完成
- 重点沉淀了哪些内容
- 推荐下一个学习的单元

## 注意事项

- 整个过程在一个对话中完成，不需要用户调第二次
- 如果对话太长导致上下文不够，提示用户先沉淀当前进度，下次再继续
- 语气要像一个耐心的学习伙伴，不要像考试官
